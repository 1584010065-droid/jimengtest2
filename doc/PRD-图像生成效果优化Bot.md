# 产品需求文档（PRD）

## 图像生成效果优化 Bot

***

## 1. 产品概述

### 1.1 产品定位

一款专注于**AI图像生成效果分析与优化**的智能对话Bot，帮助用户分析即梦等图片生成平台的生成结果，识别问题并提供专业的提示词优化方案。

### 1.2 核心价值

- **智能场景识别**：自动判断用户的图像生成场景类型
- **专业问题分析**：多维度分析生成图的优缺点
- **提示词优化**：系统性、结构性优化用户提示词
- **可复用方案**：一套通用逻辑适配多种场景

### 1.3 目标用户

| 用户类型         | 特征             | 使用场景           | 需求差异                        |
| ------------ | -------------- | -------------- | --------------------------- |
| **普通用户/爱好者** | 非专业设计背景，追求简单易懂 | 日常图片生成、社交分享    | 需要通俗易懂的解释，避免专业术语            |
| **设计师**      | 专业设计背景，对视觉敏感   | 商业设计、海报制作、创意工作 | 需要专业级分析，特别是案例3（同风格改编）这类专业场景 |

**说明**：设计师是核心用户群体之一，因为案例3（同风格改编/海报设计）这类场景主要出现在专业设计工作中，普通用户较少涉及。产品需要同时满足两类用户的需求：

- 对普通用户：提供简单直观的分析和建议
- 对设计师：提供专业深度的分析和可落地的优化方案

## 2. 核心功能需求

### 2.1 场景类型识别（核心能力）

Bot必须能够识别以下图像生成场景：

| 场景类型      | 定义             | 典型特征                 |
| --------- | -------------- | -------------------- |
| **风格转绘**  | 保持主体不变，改变视觉风格  | 有参考图，用户希望改变材质/风格     |
| **主体替换**  | 保持场景/风格不变，替换主体 | 有参考图，用户描述新主体         |
| **同风格改编** | 参考原图风格，生成新内容   | 有参考图，用户要求"同款"、"类似风格" |
| **纯文本生成** | 无参考图，纯文本描述     | 无参考图，仅有提示词           |

**场景识别逻辑**：

1. 检测是否有参考图上传
2. 分析用户提示词关键词（"变成"、"改为"、"同款"、"类似"等）
3. 对比参考图与生成图的差异
4. 输出场景类型判断及置信度

### 2.2 多维度图像分析

针对生成图进行以下维度的专业分析：

#### 2.2.1 构图与布局

- 画面结构是否合理
- 主体位置与比例
- 视觉平衡性
- 留白与空间感

#### 2.2.2 色彩与光影

- 配色协调性
- 光影效果自然度
- 氛围一致性
- 色彩饱和度/对比度

#### 2.2.3 细节与质感

- 纹理细节丰富度
- 材质表现真实感
- 边缘清晰度
- 噪点/伪影情况

#### 2.2.4 风格一致性

- 是否符合参考图风格
- 风格元素完整性
- 整体视觉统一性

### 2.3 提示词优化引擎

#### 2.3.1 提示词结构分析

- 主体描述完整性
- 场景/环境描述
- 风格/材质关键词
- 光影/氛围描述
- 技术参数（分辨率、比例等）

#### 2.3.2 优化策略

- **补充缺失要素**：添加遗漏的关键描述
- **强化风格词**：增强风格表现力
- **优化描述顺序**：按重要性重排提示词
- **添加负向提示**：排除不想要的效果
- **参数调优建议**：推荐合适的生成参数

### 2.4 输出格式

标准输出包含三个部分：

```
📊 【分析结果】
- 场景类型识别：[类型]（置信度：X%）
- 主要问题：
  1. [问题1]
  2. [问题2]
  ...
- 优点：
  1. [优点1]
  ...

💡 【解决方案】
- 问题1解决方案：[具体建议]
- 问题2解决方案：[具体建议]
- 优化方向：[整体建议]

📝 【优化后提示词】
[完整优化后的提示词]

⚙️ 【推荐参数】
- 模型版本：[建议]
- 分辨率：[建议]
- 其他参数：[建议]
```

3\. 交互设计

### 3.1 对话流程

#### 首次交互流程

```
用户上传：参考图 + 生成图 + 提示词
    ↓
Bot识别场景类型
    ↓
Bot进行多维度分析
    ↓
Bot输出完整分析报告
    ↓
用户可选择追问/优化
```

#### 追问交互

- 支持针对某个具体问题的深入询问
- 支持要求重新生成优化提示词
- 支持调整优化方向（如：更偏向写实/更偏向艺术化）

#### 分析结果卡片设计

- **场景标签**：彩色标签展示识别出的场景类型
- **折叠面板**：各分析维度可展开/收起
- **图片对比**：支持左右滑动对比参考图与生成图
- **一键复制**：优化后的提示词支持一键复制

### 3.3 输入支持

| 输入类型 | 支持情况 | 说明        |
| ---- | ---- | --------- |
| 参考图  | ✅ 可选 | 用于风格/场景参考 |
| 生成图  | ✅ 必需 | 需要分析的生成结果 |
| 提示词  | ✅ 必需 | 原始生成提示词   |
| 补充说明 | ✅ 可选 | 用户的额外需求描述 |

## 4. 测试案例覆盖

### 4.1 案例1：风格转绘（图标→糖果材质）

**输入**：

- 参考图：抖音图标（黑色）
- 生成图：彩色糖果风格图标
- 提示词："将图标材质变为糖果材质"

**预期识别**：风格转绘场景

**分析重点**：

- 材质转换的完整度
- 色彩过渡自然度
- 糖果质感的表现力
- 原图标特征的保留

### 4.2 案例2：主体替换（猫→男孩+羊群）

**输入**：

- 参考图：猫在草地上（清新漫画风）
- 生成图：男孩走在草地上，身旁有羊群
- 提示词："参考这张图片，生成场景：一个穿白衬衫的男孩走在草地上，他身旁有零零散散的羊群"

**预期识别**：主体替换场景

**分析重点**：

- 风格一致性（是否保持清新漫画风）
- 主体替换的自然度
- 场景元素的协调性
- 羊群分布的合理性

### 4.3 案例3：同风格改编（东京→上海海报）

**输入**：

- 参考图：东京主题海报（城市+东京塔）
- 生成图：上海主题海报（城市+东方明珠）
- 提示词："生成上海主题的同款海报"

**预期识别**：同风格改编场景

**分析重点**：

- 海报风格的一致性
- 构图结构的相似度
- 色彩氛围的统一性
- 地标元素的突出程度

## 5. 技术要求

### 5.1 AI能力需求

| 能力        | 用途           | 优先级 |
| --------- | ------------ | --- |
| 图像理解（VLM） | 分析图片内容、风格、问题 | P0  |
| 场景分类      | 识别生成场景类型     | P0  |
| 提示词生成     | 优化和重构提示词     | P0  |
| 多轮对话      | 支持追问和迭代      | P1  |

### 5.2 推荐模型

- **图像分析**：豆包VLM / GPT-4V / Claude 3
- **提示词优化**：GPT-4 / Claude 3.5 / 豆包大模型
- **场景识别**：可训练轻量级分类器或使用LLM判断

### 5.3 性能要求

- **响应时间**：单次分析 < 10秒
- **并发支持**：支持多用户同时使用
- **图片处理**：支持常见格式（JPG/PNG/WebP），单张<10MB

## 6. Prompt设计要点

### 6.1 系统Prompt核心要素

```
你是专业的AI图像生成优化助手，擅长分析即梦等平台的生成结果。

【核心能力】
1. 场景识别：准确判断用户的图像生成场景类型
2. 问题分析：从构图、色彩、细节、风格四个维度分析问题
3. 提示词优化：系统性优化提示词，提升生成质量

【分析框架】
- 构图与布局：画面结构、主体位置、视觉平衡
- 色彩与光影：配色协调、光影自然、氛围一致
- 细节与质感：纹理丰富度、材质真实感、边缘清晰
- 风格一致性：是否符合参考图风格

【输出要求】
- 使用中文回复
- 结构清晰，分点说明
- 提供可直接使用的优化后提示词
- 给出具体的参数建议

【场景类型定义】
1. 风格转绘：保持主体，改变风格/材质
2. 主体替换：保持场景风格，替换主体
3. 同风格改编：参考风格，生成新内容
4. 纯文本生成：无参考图，纯描述生成
```

### 6.2 场景识别Prompt

```
请分析用户的输入，判断这是哪种图像生成场景：

输入信息：
- 是否有参考图：{has_reference}
- 用户提示词：{prompt}
- 参考图描述：{reference_desc}
- 生成图描述：{generated_desc}

请判断场景类型（风格转绘/主体替换/同风格改编/纯文本生成），并说明判断依据。
```

### 6.3 图像分析Prompt

```
请对这张生成图进行专业分析：

【场景类型】{scene_type}
【用户原始提示词】{original_prompt}
【参考图】{reference_image}
【生成图】{generated_image}

请从以下维度分析：
1. 构图与布局
2. 色彩与光影
3. 细节与质感
4. 风格一致性（如有参考图）

对每个维度给出：
- 评分（1-5分）
- 优点
- 存在的问题
- 改进建议
```

## 7. 成功指标

### 7.1 功能指标

| 指标      | 目标值  | 说明             |
| ------- | ---- | -------------- |
| 场景识别准确率 | >90% | 正确识别四种场景类型     |

| 指标    | 目标值  | 说明            |
